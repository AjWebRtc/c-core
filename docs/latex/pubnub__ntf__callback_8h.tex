\subsection{Detailed Description}
This is the \char`\"{}callback\char`\"{} notification interface.

Pubnub client library offers two \char`\"{}mostly portable\char`\"{} interfaces for getting notification about the outcome of a Pubnub transaction. Those are\-:
\begin{DoxyItemize}
\item sync (this one)
\item callback
\end{DoxyItemize}

There are others, which are specific to a platform (like process events for Contiki O\-S). Also, not all platforms support the two \char`\"{}mostly portable\char`\"{} interfaces mentioned above.

The \char`\"{}callback\char`\"{} interface provides a means to register a callback function to be called with the outcome of a Pubnub transaction. There is only one callback function per a Pubnub context.

The user should not assume anything about the thread on which the callback is called. It is up to the user to synchronize access to the Pubnub context for which the callback was called with any (other) threads that might be trying to use it at the same time.

Registers a callback function to be called when a transaction ends. While it is O\-K to register a N\-U\-L\-L pointer, which means no callback will be called, it is useful only in very specific circumstances. Also, {\ttfamily N\-U\-L\-L} is the initial value (after calling \hyperlink{pubnub__coreapi_8h_af3efadf99ac4b832949265a675ec87b0}{pubnub\-\_\-init()}), so no need to set it.

Don't make any assumptions about the thread on which this function is called.
