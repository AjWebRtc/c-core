\subsection{Detailed Description}
This is the \char`\"{}sync\char`\"{} notification interface.

Pubnub client library offers two \char`\"{}mostly portable\char`\"{} interfaces for getting notification about the outcome of a Pubnub transaction. Those are\-:
\begin{DoxyItemize}
\item sync (this one)
\item callback
\end{DoxyItemize}

There are others, which are specific to a platform (like process events for Contiki O\-S). Also, not all platforms support the two \char`\"{}mostly portable\char`\"{} interfaces mentioned above.

Sync interfaces is the simplest. You just check for the (last) result of a context (by using \hyperlink{pubnub__coreapi_8h_a5cea75959cd319da41900771686f4956}{pubnub\-\_\-last\-\_\-result()}) and when it becomes something other than \#\-P\-N\-R\-\_\-\-S\-T\-A\-R\-T\-E\-D, you have your outcome.

Since \hyperlink{pubnub__coreapi_8h_a5cea75959cd319da41900771686f4956}{pubnub\-\_\-last\-\_\-result()} is part of the \char`\"{}\-Core A\-P\-I\char`\"{}, this interface only provides one additional, helper function, which will wait in a loop for the transaction to finish. While \hyperlink{pubnub__coreapi_8h_a5cea75959cd319da41900771686f4956}{pubnub\-\_\-last\-\_\-result()} is available in all Pubnub client libraries, \hyperlink{pubnub__ntf__sync_8h_ad5f9dcf61e6c902e08a12204cebddae9}{pubnub\-\_\-await()} is not.
